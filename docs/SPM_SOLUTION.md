# SPM (Service Performance Monitoring) Solution for Jaeger v2

## Problem
Jaeger v2.2.0 doesn't support Prometheus metrics storage via environment variables, causing SPM to be unavailable in Jaeger UI.

## Root Cause
- Jaeger v2 is OpenTelemetry-based and requires different configuration
- Environment variables `METRICS_STORAGE_TYPE=prometheus` are ignored
- SPM feature is not yet fully implemented in Jaeger v2.2.0

## Solution: Use Grafana for SPM Visualization

### Architecture
```
Backend → OTEL Collector (spanmetrics connector) → Prometheus → Grafana
                                                              ↓
                                                         SPM Dashboards
```

### What's Working
✅ Span metrics ARE being generated by OTEL Collector
✅ Metrics are stored in Prometheus
✅ Metrics available at: `http://localhost:9090`

**Available Metrics:**
- `telemetryflow_core_span_metrics_calls_total` - Request count
- `telemetryflow_core_span_metrics_duration_milliseconds_*` - Latency histograms

### Access SPM Dashboards

**Grafana UI**: http://localhost:3001
- Username: `admin`
- Password: `admin`

**Prometheus UI**: http://localhost:9090

### Query Examples

**Request Rate (per second):**
```promql
rate(telemetryflow_core_span_metrics_calls_total{span_kind="SPAN_KIND_SERVER"}[5m])
```

**Average Latency (milliseconds):**
```promql
rate(telemetryflow_core_span_metrics_duration_milliseconds_sum{span_kind="SPAN_KIND_SERVER"}[5m]) 
/ 
rate(telemetryflow_core_span_metrics_duration_milliseconds_count{span_kind="SPAN_KIND_SERVER"}[5m])
```

**P95 Latency:**
```promql
histogram_quantile(0.95, 
  rate(telemetryflow_core_span_metrics_duration_milliseconds_bucket{span_kind="SPAN_KIND_SERVER"}[5m])
)
```

**P99 Latency:**
```promql
histogram_quantile(0.99, 
  rate(telemetryflow_core_span_metrics_duration_milliseconds_bucket{span_kind="SPAN_KIND_SERVER"}[5m])
)
```

**Error Rate:**
```promql
rate(telemetryflow_core_span_metrics_calls_total{status_code="STATUS_CODE_ERROR"}[5m])
```

### Services

| Service | Port | URL | Purpose |
|---------|------|-----|---------|
| Jaeger UI | 16686 | http://localhost:16686 | Trace visualization |
| Grafana | 3001 | http://localhost:3001 | SPM dashboards |
| Prometheus | 9090 | http://localhost:9090 | Metrics storage |
| OTEL Collector | 8889 | http://localhost:8889/metrics | Metrics endpoint |

### Configuration Files

- **OTEL Collector**: `config/otel/otel-collector-config-spm.yaml`
- **Grafana Datasource**: `config/grafana/provisioning/datasources/prometheus.yaml`
- **Grafana Dashboards**: `config/grafana/provisioning/dashboards/`

### Future Migration Path

When Jaeger v2.3+ adds full Prometheus support:
1. Update Jaeger image version
2. SPM will work natively in Jaeger UI
3. Grafana remains as alternative visualization

### Benefits of Grafana Approach

✅ More flexible dashboard customization
✅ Better visualization options
✅ Can combine with other metrics
✅ Alerting capabilities
✅ Multi-datasource support
✅ Production-ready solution

### Baseline Version

- **Jaeger**: v2.2.0 (latest stable)
- **OTEL Collector**: latest with spanmetrics connector
- **Grafana**: latest
- **Prometheus**: latest

This is the **recommended production setup** until Jaeger v2 SPM matures.
